HelioCloud User Portal Quickstart

Welcome to the HelioCloud AWS User Portal! This portal is a way for you to monitor your EC2 instances, key pairs, and accumulated spending.

Launch an Instance

An instance is a virtual server in Amazon's Elastic Compute Cloud (EC2). You can launch an instance either from the home page or from the "Launch an Instance" page. You can choose from different images (operating systems) and types (compute power and size) according to your needs. You can also specify how much volume storage you want to attach to your instance (default 8 GB). Be sure to attach a key pair that you have access to, or create a new key pair, in order to connect to it via SSH.

Once an instance is launched, it can be tracked under the "Running Instances" section on the home page. To SSH into an instance, use the "Connect via SSH" command in the same directory that your private key is stored. You can have 5 running instances at a time.

When you have finished using your instance, be sure to click "Stop Instance" so that you do not waste computing resources and money. Like shutting down a computer, stopped instances can be restarted, and the instance will be restored to the state it was in when it was stopped. Stopped instances will appear in the "Previously Stopped Instances" section on the home page. To restart an instance, click on "Actions >> Start Instance" under the instance you want to restart.

If you no longer need an instance, you can delete that instance. Make sure that instance is stopped, then click on "Actions >> Terminate Instance" under the instance you want to delete. A deleted instance cannot be restored.

It is always best practice to launch an instance with only as much compute power and storage that you need. Click here to learn more about choosing the most appropriate instance type for your application.

Managing Key Pairs

A key pair is a set of security credentials consisting of a public key and private key that you use to prove your identity when connecting to an AWS EC2 instance. AWS stores the public key and you store the private key. This allows you to securely SSH into your instance. When creating a key pair, download and save the private key file on your computer. You will not be able to download the private key again. If you lose the private key to a key pair, delete the key pair and create a new one.

You can generate new keys and see a list of existing key pairs on through the "Manage Key Pairs" page. Once you download your private key, change the permissions on the private key file like so: `chmod 400 {keypair_name}.pem`. To SSH into an instance using your private key, make sure you are in the same directory as your private key and use the "Connect via SSH" command provided for your instance on the home page.

The portal will allow you to manage 5 active key pairs at a time. If you no longer need a key pair, or lose access to the private key of a pair, please delete the key pair by clicking "Delete" next to the key pair name.

Monitoring Accumulated Spending

AWS resources cost money, and the accumulated and weekly spending tabs on the home are a reminder of that. The costs are summed over all resources tagged with your username, regardless of if the resource was launched from this portal. Costs are not real time: your usage today will show up in your accumulated spending amount tomorrow.
